Finding the Highest Mountain Peak in a Terrain Map
A group of explorers is analyzing a terrain map represented as a 1D array of elevations. Each integer in the array represents the altitude of a specific location. A peak is a location where the altitude is strictly greater than its neighbors.

The explorers need your help to identify a peak location efficiently, as the dataset is very large. If there are multiple peaks, return any one of them.

To ensure fast analysis, you must write an O(log n) algorithm using binary search.



Input Format

The first line contains an integer n, the number of elevation points in the terrain.
The second line contains n space-separated integers representing altitudes.


Constraints

1 ≤ n ≤ 10^5
-10^6 ≤ altitude[i] ≤ 10^6
altitude[-1] = altitude[n] = -∞ (imaginary out-of-bounds values are considered as negative infinity).


Output Format

Print one peak's index (0-based). If multiple peaks exist, return any of them.


Example 1

Sample Input 1

4 

1 2 3 1 



Sample Output 1

2



Explanation

The elevation 3 at index 2 is higher than its neighbors 2 and 1, making it a peak.



Example 2

Sample Input 2

7 

1 2 1 3 5 6 4 



Sample Output 2

5



Explanation

There are two peaks:

2 at index 1 (greater than 1 and 1)
6 at index 5 (greater than 5 and 4)
The algorithm can return either 1 or 5.
